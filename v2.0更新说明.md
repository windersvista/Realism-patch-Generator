# v2.0 更新说明 - ItemToClone 格式支持

## 更新日期
2024年（最新版本）

## 主要更新

### 1. 新增 ItemToClone 格式支持
- **新增格式类型**: 支持 Items 文件夹中使用的 `ItemToClone` 格式
- **格式特征**:
  ```json
  {
    "item_id": {
      "ItemToClone": "CONSTANT_NAME",
      "OverrideProperties": {...},
      "LocalePush": {...},
      "Handbook": {...}
    }
  }
  ```

### 2. HandbookParent 到 parent_id 映射
新增 `HANDBOOK_PARENT_TO_ID` 映射表，支持从 Handbook 字段推断物品类型：

**武器类型**:
- AssaultRifles, AssaultCarbines, Handguns
- MachineGuns, MarksmanRifles, SniperRifles
- Shotguns, SMGs, GrenadeLaunchers

**配件类型**:
- Magazines, Sights, Scopes, IronSights
- Stocks, Handguards, Barrels
- Suppressors, Flashhiders, Grips
- PistolGrips, Mounts, Receivers
- ChargingHandles, GasBlocks

**装备类型**:
- Armor, Backpacks, ChestRigs
- Headwear, FaceCover, Headphones

**其他类型**:
- Containers（容器）
- MechanicalKeys, Keycards（钥匙）
- Info（信息物品）

### 3. ItemToClone 常量前缀智能识别
支持通过常量名称前缀自动推断物品类型：

**武器前缀**:
- `ASSAULTRIFLE_` → 突击步枪
- `RIFLE_`, `MARKSMANRIFLE_` → 精确射手步枪
- `SNIPER`, `SNIPERRIFLE_` → 狙击步枪
- `SHOTGUN_` → 霰弹枪
- `SMG_` → 冲锋枪
- `PISTOL_`, `HANDGUN_` → 手枪
- `MACHINEGUN_` → 机枪
- `GRENADELAUNCHER_` → 榴弹发射器

**配件前缀**:
- `MAGAZINE_`, `MAG_` → 弹匣
- `RECEIVER_` → 机匣
- `BARREL_` → 枪管
- `STOCK_` → 枪托
- `HANDGUARD_` → 护木
- `GRIP_`, `FOREGRIP_` → 前握把
- `PISTOLGRIP_` → 手枪握把
- `SIGHT_`, `SCOPE_` → 瞄具
- `SILENCER_`, `SUPPRESSOR_` → 消音器
- `FLASHHIDER_`, `MUZZLE_` → 制退器
- `MOUNT_` → 导轨

**装备前缀**:
- `ARMOR_`, `VEST_` → 护甲
- `HEADWEAR_`, `HELMET_` → 头盔
- `HEADPHONES_` → 耳机
- `FACECOVER_` → 面罩

**其他前缀**:
- `AMMO_` → 弹药
- `CONTAINER_`, `SECURE_` → 容器
- `KEY_`, `KEYCARD_` → 钥匙
- `INFO_`, `DIARY_` → 信息物品

### 4. HandbookParent 直接 ID 支持
支持 HandbookParent 字段直接使用 24 字符的物品 ID（如 `"5b47574386f77428ca22b341"`）

## 技术改进

### 代码结构优化
1. **detect_item_format()**: 新增 ITEMTOCLONE 格式检测
2. **extract_item_info()**: 添加 ITEMTOCLONE 格式信息提取
3. **process_single_item()**: 实现完整的 ITEMTOCLONE 处理流程

### 兼容性
- 完全向后兼容 v1.x 版本的所有格式
- 支持格式: TEMPLATE_ID, VIR, CLONE, STANDARD, ITEMTOCLONE
- 递归文件夹扫描保持不变（v1.9 功能）

## 使用方法

1. 将 MOD 物品数据 JSON 文件放入 `Items` 文件夹（支持子文件夹）
2. 运行 `运行补丁生成器.bat` 或直接执行 `python generate_realism_patch.py`
3. 在 `output` 文件夹中查看生成的补丁文件

## 已知限制

1. **物品分类**: 容器、钥匙、特殊物品等会被归类为"配件"（WeaponMod），这是由于现实主义MOD的类型系统限制
2. **模板缺失**: 对于 ItemToClone 引用的物品，如果在模板库中找不到对应模板，会使用默认模板
3. **Clothes 支持**: Clothes（服装）类型物品暂不支持（需要额外的类型映射）
