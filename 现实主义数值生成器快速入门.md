# 现实主义数值生成器快速入门（v3.12）

## 3 步完成一次生成

### 第 1 步：准备输入

将要处理的 JSON 放入 `input/`（支持子目录）。

支持的物品格式：

- `CURRENT_PATCH`
- `STANDARD`
- `CLONE`
- `ITEMTOCLONE`
- `VIR`
- `TEMPLATE_ID`

### 第 2 步：运行生成器

Windows 直接双击：

- `现实主义数值生成器.bat`

或命令行运行：

```powershell
.\.venv\Scripts\python.exe generate_realism_patch.py
```

### 第 3 步：查看输出

结果写入 `output/`，并保持与 `input/` 相同的目录结构。

命名规则：

- 默认：`原文件名_realism_patch.json`
- 当某输入文件内 `CURRENT_PATCH` 占比大于 50%：输出保持原名 `原文件名.json`

示例：

- 输入：`input/attatchments/ScopeTemplates.json`
- 输出（常规）：`output/attatchments/ScopeTemplates_realism_patch.json`
- 输出（CURRENT_PATCH 占多数时）：`output/attatchments/ScopeTemplates.json`

## 运行后应看到什么

控制台会输出：

- 处理文件进度
- 跳过原因（如模板缺失、格式不可识别）
- 分类统计（武器/配件/子弹/装备/消耗品）
- 每个源文件的导出路径

## 30 秒自检清单

1. `output/` 已被刷新（旧文件会清理）。
2. 关键输入文件都对应有输出文件。
3. 输出 JSON 能正常打开且结构完整。
4. 目标字段（如后坐、人机、穿深）落在规则区间内。

## 常见问题

Q：终端出现“跳过某物品”？  
A：先看提示原因。常见是 `TemplateID` 找不到模板、输入字段不足、或格式不匹配。

Q：为什么有些文件名带 `_realism_patch`，有些没有？  
A：这是当前输出策略：当文件内 `CURRENT_PATCH` 占多数（>50%）时，保持原文件名。

Q：想改数值风格，改哪里最快？  
A：优先改这四个规则文件，然后重新生成：

- `weapon_rule_ranges.py`
- `weapon_refinement_rules.py`
- `attachment_rule_ranges.py`
- `ammo_rule_ranges.py`
