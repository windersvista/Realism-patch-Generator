# 高级配置指南（v3.12）

本文档面向需要深度定制生成逻辑的用户，重点说明规则文件、识别链路和常见高级改法。

## 1. 优先改哪里

建议优先改独立规则文件，而不是直接改主脚本：

- `weapon_rule_ranges.py`：武器第一层基础区间。
- `weapon_refinement_rules.py`：武器第二层修正（口径 + 枪托形态）与口径关键词映射。
- `attachment_rule_ranges.py`：附件档位区间。
- `ammo_rule_ranges.py`：弹药三层规则（口径 + 穿深 + 弹种）。

改完后直接重新运行生成器即可生效。

## 2. 什么时候改主脚本

仅当以下场景出现时建议改 `generate_realism_patch.py`：

- 需要新增 parentId 到模板文件映射（`PARENT_ID_TO_TEMPLATE`）。
- 需要新增字符串 parentId 名称映射（`ITEM_TYPE_NAME_TO_ID`）。
- 需要改变格式识别逻辑（如新增一种输入结构）。
- 需要改变输出策略（如 CURRENT_PATCH 占比阈值）。

## 3. 输出策略高级说明

当前版本只按源文件导出，并保留 `input/` 目录结构：

- 默认输出：`原文件名_realism_patch.json`
- 当文件内 CURRENT_PATCH 占比 > 50%：输出保持原文件名 `原文件名.json`

说明：运行时会先清空 `output/`，再导出新结果。

## 4. 常用高级调参

### 4.1 武器二级修正

在 `weapon_refinement_rules.py` 中：

- 调 `WEAPON_CALIBER_RULE_MODIFIERS`：改变不同口径风格差异。
- 调 `WEAPON_STOCK_RULE_MODIFIERS`：改变枪托形态差异。
- 调 `CALIBER_PROFILE_KEYWORDS`：补充口径识别关键词（如新口径命名）。

### 4.2 附件分档命中率

在 `attachment_rule_ranges.py` 配完区间后，若发现某类附件未命中：

- 先检查输入里 `parentId`、`ModType`、`Name` 是否合理。
- 再检查主脚本中的名称兜底和模板兜底逻辑（瞄具与弹匣最常见）。

### 4.3 弹药层级差异

在 `ammo_rule_ranges.py`：

- 第一层：`AMMO_PROFILE_RANGES` 决定口径基础风格。
- 第二层：`AMMO_PENETRATION_TIERS` + `AMMO_PENETRATION_MODIFIERS` 决定穿深分层偏移。
- 第三层：`AMMO_SPECIAL_KEYWORDS` + `AMMO_SPECIAL_MODIFIERS` 决定型号差异（AP、Tracer、HP 等）。

## 5. 手动微调输出（源文件模式）

当前不再建议按固定聚合文件（如 weapons_realism_patch.json）手改。

推荐流程：

1. 在 `output/` 下找到对应输入源文件的输出 JSON。
2. 只改目标 ItemID 的字段。
3. 回写到你自己的补丁工程。
4. 如果该调整具有普遍性，回填到规则文件，避免下次被覆盖。

## 6. 常见高级问题

Q：为什么有的附件会生成 Handling 字段，即使输入没有？  
A：附件规则应用启用了 `ensure_fields=True`，规则字段缺失时会先补种子值再采样。

Q：ScopeTemplates 里很多瞄具都偏向红点档，正常吗？  
A：是设计如此。`ModType=sight` 且关键词未命中时优先红点兜底，避免高倍率误判扩散。

Q：CURRENT_PATCH 文件为什么输出不带后缀？  
A：当该文件中 CURRENT_PATCH 占比超过 50%，输出采用原文件名模式。

## 7. 建议的变更流程

1. 先用小样本输入验证规则修改。
2. 对照三份规则指南检查区间是否一致。
3. 再跑完整输入集。
4. 记录变更点到 `CHANGELOG.md`。
